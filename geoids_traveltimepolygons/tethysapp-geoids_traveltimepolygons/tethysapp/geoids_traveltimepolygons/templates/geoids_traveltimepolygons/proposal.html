Project Goal:

Travel Time Calculator. This application would calculate travel times in a vehicle, on bike, or on foot showing the user a radius of where they could get to in a specified time. This solves a traffic engineering problem that includes calculating FFS, LOS, distance, and temporal variation of traffic. We could add to this, for example it could calculate the population living within a certain travel time to predict trips that could be attracted to an area, or it could show an elevation profile along a line drawn to help walkers/runners/bikers pick the best 10 minute path, it could populate a list of known places/locations within a given time like stores, government buildings etc.

From an engineering standpoint this could be a critical tool for transportation planners and traffic engineers to get instant feedback on the market area of a new development. It could be used as a PR tool to help the community see what is available to them nearby. Typically this kind of analysis is run on complex GIS modeling software and is available to very few people. We believe that if this information could be more readily shared across the region and help decision makers to make informed decision that will improve the transportation infrastructure of the city.



Required Vector and Raster Analysis: Some sort of network analysis will be required for the vector data, which consists of roads. Travel times for bicycles could be estimated using raster analysis when considering slope.

    • Geoprocessing Workflow: Calculate travel times based on speeds from existing traffic conditions and speed limits and use the network analyst tool to create a series of polygons that connect routes based on travel times. Travel times for bicycles could be calculated factoring in slope, which would use the slope tool with some DEM data.



    User Interface:

    1. User Picks a starting point (by clicking on the map, typing in an address, or coordinates)
    2. User selects mode of transportation
    3. User Selects the time of day they would like to travel (this wouldnâ€™t affect walking or biking so much but may have an impact on driving so maybe this option is only available for drivers)
    4. User selects one or more time periods they would like to look at (10, 15, 30 min etc)
    5. User clicks display/calculate
    6. The app creates a colored shapefile for each selected time period. Each shape file is centered at the starting point and shows the boundary in every direction for how far that person could get in the given time.
